# Модернизация ETL‑пайплайна и построение витрины для анализа возвращаемости клиентов
Цель проекта: адаптировать существующий ETL‑пайплайн под новые бизнес‑требования (учёт статусов заказов shipped и refunded) и реализовать витрину для расчёта метрик возвращаемости клиентов (customer retention) в недельной разбивке — с обеспечением обратной совместимости и корректного учёта возвратов в финансовых показателях.

Задачи проекта:

Проанализировать новые вводные от заказчика: спецификацию API с инкрементальными данными о продажах и статусами заказов (shipped, refunded), а также требования к витрине.

Доработать ETL‑пайплайн с учётом новых статусов: обеспечить корректную обработку refunded, гарантировать, что все данные в витрине учитываются как shipped, и сохранить backward compatibility с предыдущими версиями пайплайна.

Реализовать логику расчёта метрик для витрины: определить и рассчитать показатели возвращаемости клиентов в разрезе недель (новые, возвращающиеся и клиенты с возвратами), а также соответствующие финансовые метрики (выручка, суммы возвратов) для каждого сегмента.

Интегрировать новые преобразования в единый пайплайн, протестировать корректность расчётов и убедиться, что итоговый код покрывает все сценарии, описанные в двух этапах развития пайплайна.

Используемые инструменты:

Apache Airflow (оркестрация ETL‑процессов)

Docker (контейнеризация пайплайна)

SQL (трансформации и расчёт метрик в витринах)

PostgreSQL (целевое хранилище данных)

Вывод:
В результате выполнения проекта ETL‑пайплайн был успешно адаптирован под новые бизнес‑правила: обеспечена корректная обработка статусов shipped и refunded в витрине с сохранением обратной совместимости. Реализована витрина customer_retention, позволяющая анализировать возвращаемость клиентов в недельной разбивке и оценивать влияние возвратов на финансовые показатели по сегментам. Итоговый код включает решения для всех сценариев, протестирован и готов к эксплуатации. Результаты проекта можно считать удовлетворительными.
